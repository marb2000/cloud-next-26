# MemoTattoo - Cloud Next 2026 Showcase

## Overview

MemoTattoo is a full-stack platform demonstrating a state-of-the-art implementation of Firebase products , showcased at Cloud Next 2026. The project encompasses a rich Android mobile application, a powerful Web Admin Console, and a highly automated backend infrastructure.

The platform features a unique **Flashcard Studio** that leverages AI for image generation and refinement, a custom economy system, and advanced automated testing strategies.

---

## Key Features

### 🎨 Flashcard Studio
- **5-Step Creation Wizard**: A guided experience for users to create custom flashcards.
- **Iterative AI Image Refinement**: Uses **Nano Banana** for multimodal handling to generate and refine images based on user prompts.
- **Dynamic UI**: Features high-resilience Android game sessions and adaptive UI components.

### 💰 Platform Economy
- **'Bolts' Economy**: A tiered virtual currency system that drives user engagement and content creation.

### 📱 Android Application
- **Jetpack Compose UI**: Built entirely with modern Jetpack Compose for a smooth and reactive interface.
- **Advanced UI Patterns**: Utilizes Modal Bottom Sheets for contextual actions and adaptive badge rows.
- **Performance Optimized**: Implements state recycling in long lists (`LazyColumns`) and dynamic text scaling (`AutoResizeText`).

### 🖥️ Admin Console
- **Moderation Queue**: Features a zoom inspector for detailed review of generated content.
- **Design System**: A custom design system applied across the admin interface for a premium look and feel.

### 🤖 Firebase AI Logic
- **Server Prompt Templates**: Implements server-side prompt templates for AI interactions, ensuring consistency and security.

## Setup and Provisioning

MemoTattoo includes a comprehensive set of scripts to automate the setup of the Firebase environment. These are located in `memotattoo/backend/scripts/setup/`.

### Key Scripts

1.  **`create-project.js`**: Automates the creation of the Google Cloud project and enables Firebase on it.
2.  **`setup-apps.js`**: Registers the Android and Web applications in your Firebase project.
3.  **`setup-appcheck.js`**: Configures App Check with Play Integrity (Android) and reCAPTCHA v3 (Web), and registers debug tokens.
4.  **`download-config.js`**: Fetches the generated configuration files (`google-services.json` for Android and environment variables for Web) and places them in the correct directories.
5.  **Automation & Deployment**: Includes scripts to automate all deployment and updates for the application (e.g., prompt sync, app deployment).

### Typical Workflow

1.  Configure your project details in `sync-config.json`.
2.  Run `create-project.js` to provision the project.
3.  Run `setup-apps.js` to register applications.
4.  Run `setup-appcheck.js` to secure your backend.
5.  Run `download-config.js` to pull the configuration files into the apps.
